# v0.8 trial #2 prep

Addendum to [`docs/v0.6-trial-protocol.md`](./v0.6-trial-protocol.md)
listing what's changed since trial #1 (2026-05-16) and what the
trial #2 operator should test specifically. The core protocol is
unchanged; the additions below are bonuses to verify.

## What's new since trial #1

Between trial #1 and the v0.8 launch, bosun gained:

- **v0.6.1 / v0.6.2** — per-op git timeout coverage (worktree-add, fsck, rev-parse, reset-hard, every Client method). Trial #1's blockers (silent init hang, broken --resume) are fixed.
- **v0.7** — launch UX (default brief prompt), predictor accuracy, stale-branch refusal, merge load-check, state-file phantom filter, corrupted-gitdir recovery, salvage on remove --force.
- **v0.7 bug-hunt** — claims phantom filter, suggest timeouts, proc detection via cmdline, init --force progress, MCP goroutine leak, rescue salvage error surfacing, cleanup/merge silent-error fixes.
- **v0.7 refactors** — `internal/phantom`, `internal/lockfile`.
- **v0.7 test suite** — `make fuzz`, `make stress`, fuzz tests across brief/predict/session/phantom/preflight parsers.
- **v0.8 lane 1** — `bosun doctor` for one-command preflight.
- **v0.8 lane 2** — `bosun init --suggest "<goal>"` for one-step onboarding.

## Trial #2 specifics

In addition to the standard protocol, exercise:

1. **`bosun doctor` on the trial repo before anything else.** Expected: PASS or WARN. Any FAIL = stop. Capture the output in the findings doc — this is the gate the launch checklist gives newcomers.

2. **`bosun init --suggest "<some real goal>"` instead of hand-writing a brief.** Verifies the v0.8 lane 2 path end-to-end. The proposed plan should validate cleanly; if it surfaces overlap/cycle errors, that's a real finding worth documenting.

3. **Verify `BOSUN_PROC_DEBUG=1 bosun status` does what it says.** If proc detection misfires (the v0.7 bug-hunt fix), the diagnostic should surface the candidate-skip reason. macOS-specific.

4. **Test the resilience gates explicitly.** Force-close one session's Ghostty window mid-edit; expect CRASHED + `bosun rescue` to snapshot dirty files. Try `bosun merge` with a live agent in another session; expect refusal. Run `bosun merge --undo` to verify reflog reset.

5. **Corrupted-gitdir recovery.** Outside the normal flow: rm `<trial-repo>/.git/worktrees/<bosun-session-1>/HEAD` to simulate the crash footprint. Run `bosun rescue session-1`; expect a clean refusal with a recovery hint, not a confusing mid-snapshot git error. Then `bosun remove session-1 --force`; expect salvage into `.bosun/rescues/`.
