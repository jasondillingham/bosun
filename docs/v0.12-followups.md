## Medium

### M1 — Brief size in `bosun_spawn` is unbounded — **FIXED 2026-05-21 (Bundle A)**

**Where:** `internal/mcp/tool_spawn.go` — `BriefMarkdown` arg had no size cap.
**Resolution:** `maxBriefBytes = 256 << 10` constant + refusal at the toolSpawn gate before any allocation downstream. Refused requests audit under `spawnGateInvalidArgs`. Test: `TestSpawn_OversizedBriefRefuses`.

### M2 — Anthropic API response has no size limit — **FIXED 2026-05-21 (Bundle A)**

**Where:** `internal/suggest/claude.go` — `raw, err := io.ReadAll(resp.Body)`.
**Resolution:** Wrapped in `io.LimitReader(resp.Body, maxAnthropicResponseBytes)` (10 MiB). Compromised/proxied endpoint can no longer drive unbounded allocator pressure.

### M3 — TOCTOU race in `bosun_spawn` concurrent quota — **FIXED 2026-05-21**

**Where:** `internal/mcp/tool_spawn.go:142-148`, `internal/spawn/spawn.go:150`.
**Resolution:** `spawntree.AddChildIfUnderQuota` does count+check+add atomically inside the flock; `spawn.Run` uses it. Concurrent + multi-brief callers can no longer bypass `agent_spawn.max_concurrent_sub_sessions`. Test: `TestAddChildIfUnderQuota_ConcurrentRespectsCap`.

### M4 — History archive files written with mode `0o644` — **FIXED 2026-05-21 (Bundle A)**

**Where:** `internal/history/history.go` — `brief.md`, `claims.json`, `commits.log`, `merged.txt`, `metadata.json`.
**Resolution:** All `os.WriteFile` calls now use `0o600`. `copyFileIfExists` switched from `os.Create` (0o666 + umask) to `os.OpenFile(..., 0o600)` so `brief.md` and `claims.json` also land 0o600. Test: `TestArchive_CapturesBriefClaimsCommits` now asserts each archive file's mode.

### M5 — Lock acquisition has no timeout — **FIXED 2026-05-21 (Bundle B)**

**Where:** `internal/lockfile/lockfile_unix.go` + `lockfile_windows.go` — both used blocking acquire.
**Resolution:** Replaced blocking `LOCK_EX` / `LockFileEx` calls with non-blocking equivalents (`LOCK_EX|LOCK_NB` on Unix, `LOCKFILE_FAIL_IMMEDIATELY|LOCKFILE_EXCLUSIVE_LOCK` on Windows) inside a poll loop bounded by `DefaultTimeout = 30s`. Each successful acquire stamps `pid=N ts=<rfc3339>` into the lock file inside the locked section; on timeout the waiter reads that diagnostic data back and returns `*LockTimeoutError{Path, Timeout, HolderPID, HeldFor}`. The error unwraps to `ErrLockTimeout` for sentinel-style checks. Tests: `TestWithLock_TimeoutSurfacesLockTimeoutError` (real holder + contention), `TestWithLock_TimeoutErrorMessage` (formatter shape).

### M6 — HTTP serve missing security headers + body size limit

**Where:** `internal/web/server.go`, `internal/web/handlers.go`.
**Risk:** Missing `X-Frame-Options`, `X-Content-Type-Options`, `CSP`, `MaxHeaderBytes`, and `MaxBytesReader`. Default bind is `127.0.0.1` so attack surface is small, but defense-in-depth for the malicious-browser-tab case warrants `X-Frame-Options: DENY` at minimum.
**Fix:** Middleware setting the three headers + wrapping bodies with `http.MaxBytesReader(w, r.Body, 1<<20)`.

### M7 — PIPE_BUF assumption on macOS for audit/usage lines

**Where:** `internal/usage/usage.go`, `internal/mcp/spawn_audit.go`, `internal/mcp/subtask_audit.go`.
**Risk:** PIPE_BUF is 512 bytes on macOS (vs 4096 on Linux). Atomic-append breaks for entries over 512 bytes — they can tear across process boundaries.
**Fix:** Cap marshaled lines at 480 bytes (truncate `Model` / `TurnLabel`). Add a unit test pinning encoded-length under the cap.

## Low

### L1 — Anthropic 5xx error body logged verbatim — **FIXED 2026-05-21 (Bundle A)**

**Where:** `internal/suggest/claude.go` — non-2xx path's `strings.TrimSpace(string(raw))` fallback.
**Resolution:** The non-2xx fallback no longer includes the raw body. If `apiErr.Error.Message` parses, we surface that; otherwise we surface only the status code with a note that the response had no parseable `error.message`. Decode-failure path on 2xx (separate concern) intentionally retains its 200-char truncated body since that's a debugging aid for malformed-but-legitimate responses.

### L2 — `bosun_attach` allows arbitrary PIDs

**Where:** `internal/mcp/tool_attach.go:58-59`.
**Risk:** Agent registers PID 1 / sshd / etc. as its liveness PID — bogus RUNNING claims until `proc.IsAlive` disproves them (never, for PID 1).
**Fix:** Best-effort validate that PID's working directory is inside the session's worktree. Fall back to current behavior on platforms without the check.

### L3 — Custom MCP tools inherit the bosun process env

**Where:** `internal/mcp/tools_custom.go:91` — `exec.CommandContext` with no `cmd.Env` override.
**Risk:** Operator-defined tools see the daemon's `ANTHROPIC_API_KEY` and other secrets. Documented as expected ("operator trusts their own tools") but worth surfacing in the docs.
**Fix:** Doc note in Phase 5 #61 spec. No code change unless explicit env-scrubbing per def is wanted.

### L4 — `BOSUN_BRIEF.md` not in repo's `.gitignore` — **FIXED 2026-05-21 (Bundle A)**

**Where:** repo-root `.gitignore`.
**Resolution:** Added `BOSUN_BRIEF.md` with comment explaining the per-session brief may contain inline operator context. Note: each session worktree already gets a `.git/info/exclude` entry for `BOSUN_BRIEF.md` (set by cmd_init / spawn.Run); the root `.gitignore` is belt-and-suspenders for the main worktree case.
