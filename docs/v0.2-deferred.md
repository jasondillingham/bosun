# v0.2 deferred items

Parking lot for ideas that don't belong in v0.1. Add entries here instead of
extending the v0.1 scope.

## MCP server (planned for v0.2)

Replace `.bosun/claims/` and `.bosun/state/` with structured MCP tool calls:

- `bosun_claim <paths...>` — same as the current claim CLI but as a tool call from inside a session
- `bosun_release <paths...>` — explicit release (currently happens at merge time)
- `bosun_done [message]` — replaces touching `.bosun/state/<session>.done`
- `bosun_stuck <message>` — replaces touching `.bosun/state/<session>.stuck`
- `bosun_check <paths...>` — query whether any of these paths are claimed by another session before starting work
- `bosun_announce <event>` — broadcast something to the operator

The workflow shape stays identical — sessions just talk through tool calls instead of touching files.

## Watch mode

`bosun status --watch` re-renders every N seconds. Not in v0.1; users can `watch bosun status` as a workaround.

## Web dashboard (v0.3)

Same data as `bosun status` exposed over HTTP with server-sent events. Reuses the shape of the existing job-hunt dashboard.

## Hooks (v0.4)

Pre-init, post-merge, etc. Configured in `.bosun/config.json`.

## Custom session names

Currently numbered `session-1`, `session-2`, etc. v0.2 will support named sessions (`bosun init auth http storage`).

## Dependency-aware dispatch

"session-2 cannot start until session-1 finishes." Currently encoded by the operator writing the plan that way; v0.2 may add explicit dependency declarations.

## Resource governance

Beyond `--isolate-cache`: scheduling test runs across sessions to avoid CPU thrashing, sharing module download caches read-only, etc.

## Auto-detection of running sessions

Detect whether a Claude Code process is actually running in a worktree, vs. the worktree being abandoned. Adds a column to `bosun status`.
