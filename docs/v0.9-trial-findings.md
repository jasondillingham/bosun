## What's not exercised

- **`bosun_spawn` MCP tool.** The trial's brief left the decision to the
  agent and the audit showed solo work was correct, so spawn never
  fired. To validate the spawn path itself we need a brief that
  genuinely needs N disjoint sub-lanes from a single top-level session.
- **PreToolUse hook (v0.9.1 #6).** The hook needs `bosun hook install`
  in `.claude/settings.json` of the worktree, which requires a Claude
  Code agent actually being launched there via `bosun launch`. This
  trial was driven by the parent Claude session over `Bash` + `Edit`
  tools rather than via `bosun launch`, so the auto-claim path wasn't
  exercised either.
- **`bosun rescue`, CRASHED state, merge --undo.** Trial #2 covered
  these; trial #3 didn't force a crash on purpose.

## Findings

1. **`--suggest` prereq is surprising on first run.** Error is clear,
   but the README quick-start should mention `ANTHROPIC_API_KEY` so a
   first-time user can decide whether to use `--suggest` or `--brief`
   before running it. **Severity: docs nit.**
2. **`bosun config set` doesn't expose `agent_spawn.*` keys.** Had to
   hand-edit `.bosun/config.json` to set `agent_spawn.enabled = true`.
   This is a regression from the "no hand-editing" config UX. The set
   command's known-keys list needs `agent_spawn.enabled`,
   `agent_spawn.max_concurrent_sub_sessions`,
   `agent_spawn.max_depth`. **Severity: real bug; v0.9.1 followup.**
3. **Brief format error message is precise and useful.** No action
   needed — it's a good example of the kind of message bosun should
   keep producing.
